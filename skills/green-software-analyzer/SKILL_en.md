# Green Software Analyzer

## Overview

This skill provides a set of tools to analyze API traffic from logs, simulate requests to collect detailed Green Software metrics, and visualize the results in interactive reports and databases. The goal is to identify and quantify the impact of design patterns aimed at reducing resource consumption (energy, bandwidth, CPU).

## Core Capabilities

### 1. Green Software Log Analysis

Processes request logs in CSV format to identify usage patterns, such as duplicate requests, heavy queries (`limit=-1`), and pagination usage. Generates a JSON report with insights and static charts that summarize traffic behavior.

-   **Script**: `scripts/analyze_logs_en.py`
-   **Input**: CSV log file (e.g., `/home/ubuntu/upload/logs.csv`)
-   **Output**: JSON results (`references/analysis_results.json`), PNG charts (`analysis/output/`), and an interactive HTML web report (`templates/index.html`).

### 2. Traffic Simulation and Metrics Collection

Performs HTTP requests based on the provided logs, measuring the exact size of the request and response payload, latency, and HTTP status. Classifies each request based on its applicability to four Green Software design patterns:

-   **Green by Default**: Requests that could be more efficient by default (e.g., without `limit=-1`, with pagination, with field selection).
-   **Just Latest Updates (Delta)**: Requests for volatile data that would benefit from sending only deltas.
-   **Wish List (Sparse Fieldsets)**: Requests that could specify fields to reduce payload.
-   **Wish Template**: Complex requests that would benefit from pre-defined response templates.

Collected data is stored in an SQLite database (or PostgreSQL, with adaptation).

-   **Script**: `scripts/traffic_simulator_en.py`
-   **Input**: CSV log file (e.g., `/home/ubuntu/upload/logs.csv`)
-   **Output**: SQLite database (`green_software_metrics_en.db`) with detailed metrics and pattern classification.

### 3. Database Data Visualization

Analyzes the database generated by the simulation to create visualizations that highlight the distribution of Green Software patterns, the relationship between response size and latency, and the potential impact of each pattern. This allows for a deep understanding of optimization opportunities.

-   **Script**: `scripts/visualize_db_data_en.py`
-   **Input**: SQLite database (`green_software_metrics_en.db`)
-   **Output**: Detailed PNG charts (`analysis/output/db_viz_en/`).

## Recommended Workflow

1.  **Prepare Logs**: Provide your logs in CSV format.
2.  **Initial Analysis**: Run `scripts/analyze_logs_en.py` for an overview of traffic and identification of basic issues.
3.  **Detailed Simulation**: Run `scripts/traffic_simulator_en.py` to simulate traffic, collect payload/latency metrics, and classify requests by Green Software patterns.
4.  **Pattern Analysis**: Run `scripts/analyze_delta_pattern_en.py` (or similar scripts for other patterns) to deepen the analysis on specific patterns.
5.  **Visualize Insights**: Run `scripts/visualize_db_data_en.py` to generate charts from the database and understand the potential impact of optimizations.
6.  **Interactive Report**: Use `templates/dashboard_en.html` to view the results interactively.

## Resources

-   **`scripts/`**: Contains Python scripts for analysis, simulation, and visualization.
-   **`references/`**: Stores JSON results from initial and pattern analysis.
-   **`templates/`**: Includes the HTML template for the interactive web report.
